Token Tool MCP

Deploy and manage compliant tokens from Claude, Cursor, or any AI agent — by typing a sentence.

Built on Bitbond Token Tool — 8,300+ deployments, CertiK-audited contracts, compliance built in.

See It in Action

"Deploy a token called Green Bond A, 1M supply on Base, with whitelist and pausable."

→ CertiK-audited ERC-20 deployed on-chain in ~30 seconds. Contract address returned. No Solidity required.

Install

Claude Desktop

Add to ~/.claude/claude_desktop_config.json:

{
  "mcpServers": {
    "token-tool": {
      "command": "npx",
      "args": ["-y", "token-tool-mcp"],
      "env": {
        "BITBOND_PRIVATE_KEY": "0x..."

    }
  }
}

How It Works

You (natural language) → MCP Client → Token Tool MCP (local)
    → Bitbond contract factory → On-chain deployment
  1. You type a prompt in your MCP client
  2. The client calls the appropriate tool on the local MCP server (stdio — nothing leaves your machine yet)
  3. The MCP server constructs a deployment transaction and submits it to Bitbond's on-chain contract factory
  4. The factory deploys a CertiK-audited ERC-20 with your parameters
  5. Contract address and tx hash are returned to your client

Your private key signs transactions locally. Read from env var, never passed as an argument, never transmitted.

Security

  • Private key stays local. Read from env var only — never logged, never transmitted
  • stdio transport. No network listener, no open ports
  • CertiK-audited contracts. Battle-tested, not generated Solidity
  • Testnet by default. Start on Sepolia — free and identical to mainnet
  • Human-in-the-loop. Enable confirmation prompts for mainnet ($299/deployment)

Title

Show HN: Token Tool MCP – Deploy CertiK-audited ERC-20 tokens from Claude/Cursor in 30 seconds

Body

MCP servers exist for payments (Stripe), trading (Alpaca), and banking (Griffin). Nothing existed for compliant token issuance — the kind with investor whitelists, transfer restrictions, emergency pauses, and on-chain legal document references.

So we built one.

Token Tool MCP wraps Bitbond's Token Tool — a platform that's handled 8,300+ token deployments since 2020, using CertiK-audited smart contracts. The MCP server exposes the same infrastructure to AI agents.

"Deploy a token called DAO Treasury, symbol DTRX, 5 million supply on Arbitrum, mintable and pausable"

The agent calls deploy_token. A CertiK-audited ERC-20 is live on-chain in ~30 seconds.

How it works under the hood

The MCP server runs locally via stdio transport. When you deploy:

  1. Constructs a deployment transaction against Bitbond's on-chain smart contract factory
  2. Your private key signs locally (read from env var — never logged, never transmitted)
  3. Signed transaction broadcast to the target chain
  4. Factory deploys a CertiK-audited ERC-20 with your parameters
  5. Contract address + tx hash returned

No data goes to Bitbond's servers. The contract factory is already deployed on each chain.

What it can do

11 tools: deploy, mint, burn, pause, unpause, transfer, get token info, estimate cost, list chains, list deployments, wallet info. Compliance flags: whitelist, blacklist, pausable, force transfer, document URI, max supply cap.

12 networks: Ethereum, Polygon, BNB, Arbitrum, Base, Optimism, Avalanche, Solana, Stellar + 3 testnets.

Technical details

  • stdio transport — Claude Desktop, Cursor, VS Code, any MCP client
  • Private key from env var, never logged or transmitted
  • $299 per mainnet deployment (same as the web UI — testnet is free)
  • MIT license

Repo: github.com/thendrix-eng/token-tool-mcp
